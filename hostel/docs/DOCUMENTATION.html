Project Overview

The Hostel Management System (HostelMS) is a robust enterprise-grade solution for managing student accommodations. The system handles everything from student registration and room allocation to complaint tracking and fee management, providing a seamless experience for both students and hostel administrators.

Technology Stack

PHP 7.4+ MySQL 8.0 Bootstrap 5.3 JavaScript ES6+ FontAwesome 6 Custom Modern CSS

System Architecture

The project is structured logically to separate administration from student access:

  • root/: Student-facing pages (Login, Registration, Dashboard, Room Booking).
  • admin/: Full administrative control panel for management.
  • includes/: Core configuration, session management, and security libraries.
  • css/ & js/: Centralized styling and frontend interactive components.

Database Schema

Table Name Core Responsibility
userregistration Primary student records, gender, and validation status.
admin Privileged access accounts for hostel administrators.
rooms Room inventory, seater capacity, and fees.
registration Transactional data linking students to specific room allocations.
complaints Student grievance repository and administrative response logs.
debtor_reports Reporting system for Block Debtors to communicate with Super Admin.

Security Framework

CSRF Protection

Every state-changing action (Update, Delete) is protected by a cryptographically secure 256-bit token stored in the session and verified upon request.

Session Binding

Sessions are bound to the user's IP Address and User Agent. If either changes during a session, the system automatically terminates the session for security.

Data Integrity

The system prevents SQL Injection via Prepared Statements across all modules and prevents XSS via aggressive output encoding.
